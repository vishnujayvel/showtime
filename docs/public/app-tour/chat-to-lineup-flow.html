How it works
1. CHAT FREELY
User talks to Claude naturally. Claude can fetch calendar, search the web, ask questions. No structured prompts. No MCP restrictions. This is just a conversation.
2. BUILD LINEUP
User clicks "Build My Lineup" or says "make my lineup." Claude generates the showtime-lineup JSON. We parse it and render the Lineup Card inline. User can keep chatting to refine.
3. GO LIVE
When happy, user clicks "We're Live!" Button only activates after a lineup exists. Transitions to the ON AIR experience with timers, beats, and the rundown bar.
KEY INSIGHT
The conversation IS the planning tool. We stop fighting Claude's behavior and let it use tools freely. The only structured moment is the lineup extraction — one prompt, clear format spec, all the context already in the conversation history. This eliminates the entire class of "JSON blob" bugs.
📅 CALENDAR INTEGRATION
Calendar events from MCP show up naturally in the conversation — Claude fetches them, presents them nicely, and the user decides what to include. No pre-injection, no prompt hacks. Calendar items in the lineup card get a 📅 marker so the user knows what came from their calendar.
