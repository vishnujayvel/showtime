# Product Spec: Fixed vs Flexible Time Windows

**Issue:** [#130](https://github.com/vishnujayvel/showtime/issues/130)
**Status:** MVP
**Last updated:** 2026-03-31

## Problem

When the user runs late on an act, Showtime pushes all subsequent acts forward sequentially. This breaks calendar-synced events — you can't move a meeting with another person just because you're behind schedule.

**Example from real usage (2026-03-31):**
- Tea Time (Aseem) at 17:00 → shifted to 17:23
- Catch up (KDR) at 18:00 → shifted to 18:23
- Wind down at 19:00 → shifted to 19:23

## Industry Patterns

| App | Philosophy | Fixed events | Flexible tasks | When running late |
|-----|-----------|--------------|----------------|-------------------|
| **Reclaim.ai** | Auto-schedule | Always busy, never move | Start as "free", flip to "busy" near deadline | Auto-reschedule flexible tasks in ~15s |
| **Motion** | Full autopilot | Immovable anchors | Auto-fill gaps | Reshuffles everything around fixed |
| **Sunsama** | Intentional | Never overlap busy events | User manually timeboxes | User decides, no auto-reshuffle |

## Showtime's Approach: "The Show Adapts, But Other People's Time is Sacred"

Showtime should follow **Sunsama's intentional philosophy** (matches ADHD-friendly, no-guilt design) with **visual drift awareness** (like Reclaim's awareness without the auto-scheduling).

### Act Types

```
┌─────────────┬──────────────────────────────────────────────┐
│ Type        │ Behavior                                     │
├─────────────┼──────────────────────────────────────────────┤
│ PINNED      │ Fixed wall-clock start time from calendar.   │
│ (calendar)  │ Never shifts. Timer shows countdown to       │
│             │ start ("starts in 12 min") or overdue        │
│             │ warning ("overdue by 5 min — wrap up!")      │
├─────────────┼──────────────────────────────────────────────┤
│ FLEXIBLE    │ Solo tasks generated by Claude. Can shift    │
│ (generated) │ when running late. Duration adjusts to fill  │
│             │ available gap before next pinned event.       │
├─────────────┼──────────────────────────────────────────────┤
│ BUFFER      │ Auto-inserted 5-min gaps between acts.       │
│ (auto)      │ First to shrink when running behind.         │
│             │ Visual: thin gray line, not a full card.     │
└─────────────┴──────────────────────────────────────────────┘
```

### Key Principles

1. **Pinned events are sacred.** They show their calendar time, always. No exceptions.
2. **Flexible acts fill gaps.** They expand or shrink to fit between pinned events.
3. **No auto-rescheduling.** Showtime shows you the reality; you decide what to do.
4. **No guilt.** If you're late, the show adapts. "Tea Time got moved up in the rundown" — never "You're 23 minutes behind."
5. **Drift is visible, not punishing.** A subtle indicator shows "Act 3 running 12 min over → Tea Time starts in 8 min"

### Data Model Change

```typescript
// src/shared/types.ts
interface Act {
  id: string
  name: string
  sketch: string
  durationMinutes: number
  status: ActStatus
  beatLocked: boolean
  order: number
  startedAt?: number
  completedAt?: number

  // NEW: Fixed vs flexible scheduling
  pinnedStartAt?: number | null   // Wall-clock ms timestamp from calendar. null = flexible
  calendarEventId?: string | null // Source calendar event ID for syncing
  isFlexible: boolean             // true = can shift, false = pinned to calendar time
}
```

### Lineup Rendering Algorithm

```
for each act in lineup:
  if act.pinnedStartAt:
    // PINNED: show at calendar time, regardless of previous act
    displayStartTime = act.pinnedStartAt
    if currentTime > pinnedStartAt and act not started:
      show "overdue" badge
    else:
      show "starts at HH:MM" with countdown
  else:
    // FLEXIBLE: calculate from previous act end
    displayStartTime = previousActEndTime + bufferMinutes
    if displayStartTime overlaps next pinned event:
      // Auto-truncate: reduce duration to fit
      maxDuration = nextPinnedStart - displayStartTime - bufferMinutes
      displayDuration = min(act.durationMinutes, maxDuration)
      if displayDuration < act.durationMinutes:
        show "truncated" indicator (e.g., strikethrough of original duration)
```

### When Running Late — Drift Behavior

**Scenario:** User is in Act 2 (Walk with Steven, flexible), timer shows 12 minutes over. Tea Time (Aseem) at 17:00 is next.

**What Showtime shows:**

```
┌─ EXERCISE | ACT 2 ─── RUNNING ──┐
│  Walk with Steven                 │
│  ⏱ 72:15 (+12 min over)          │
│                                   │
│  ⚡ Tea Time (Aseem) in 8 min     │  ← drift nudge (not guilt)
└───────────────────────────────────┘

NEXT UP:
  📌 Tea Time (Aseem) · 17:00 sharp
     "Aseem is expecting you in 8 minutes"
```

**What does NOT happen:**
- Tea Time does NOT shift to 17:12
- Catch up (KDR) does NOT shift to 18:12
- No "you're 12 minutes behind schedule" guilt message

**What the user can do:**
- **End Act** → moves to Tea Time (starts immediately since it's overdue)
- **+15m** → extends Walk, but drift nudge intensifies
- **Rest** → takes intermission, drift nudge stays
- **Director** → "Skip this act, move to Tea Time" option appears

### Notification Behavior

| Condition | Notification |
|-----------|-------------|
| 5 min before pinned event | "Tea Time with Aseem starts in 5 min" (gentle) |
| Pinned event time reached | "Tea Time is starting now" (persistent) |
| 5 min after pinned event | "Tea Time started 5 min ago — wrap up when ready" (no guilt) |
| 15 min after pinned event | Auto-enter Director Mode with "Go to Tea Time" as default |

### Edge Cases

1. **Two pinned events back-to-back** → No flexible acts between them. Show them stacked.
2. **Flexible act between pinned events doesn't fit** → Show truncated duration with indicator. "Deep Work: 15 min (was 30 min — Tea Time at 17:00)"
3. **User skips a pinned event** → Mark as skipped (gray), don't remove. "That event got cut from tonight's show."
4. **Calendar event cancelled mid-show** → Convert to flexible or remove. Show notification.
5. **All acts are flexible** → Current behavior (sequential) works fine.
6. **All acts are pinned** → Pure calendar view. Gaps between events are "intermission."

### Implementation Phases

**Phase 1: Data model + lineup rendering** (this PR)
- Add `pinnedStartAt`, `calendarEventId`, `isFlexible` to Act type
- Update lineup rendering to respect pinned times
- Update `SET_LINEUP` to mark calendar-sourced acts as pinned
- Show pinned badge (📌) on calendar acts

**Phase 2: Drift awareness**
- Countdown to next pinned event during live act
- Drift nudge in expanded view and pill view
- Auto-Director-Mode when 15 min past pinned event

**Phase 3: Notifications**
- 5-min-before notification for pinned events
- Persistent notification when pinned event starts
- Gentle reminder when overdue

## Non-Goals

- Auto-rescheduling (too opinionated for ADHD users)
- Writing back to Google Calendar
- Splitting acts automatically
- Calendar event editing from within Showtime
