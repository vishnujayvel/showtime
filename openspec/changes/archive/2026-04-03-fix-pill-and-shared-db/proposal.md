## Part C: Fixed vs Flexible Time Windows (GitHub #130) — Enhancement (MVP)

### Problem
Calendar-synced acts (meetings with other people) get pushed forward when the user runs late on a previous act. Fixed-time events must stay pinned to their calendar time.

### Design

Acts need a `pinned` flag:
- **Pinned acts** (from calendar): Display their original `plannedStartAt`, never recalculate. Show drift indicator if current time has passed them.
- **Flexible acts** (generated by Claude): Can shift based on schedule drift.

#### Data model change
```typescript
interface Act {
  // ... existing fields
  pinnedStartAt?: number | null  // Wall-clock time from calendar, null = flexible
  calendarEventId?: string | null  // Link to source calendar event
}
```

#### Lineup rendering change
When computing start times for the lineup:
- If act has `pinnedStartAt` → use it, show strikethrough of original if drifted
- If act is flexible → calculate from previous act end time
- If flexible act overlaps a pinned act → show warning, auto-truncate or prompt

#### XState context change
Add `pinnedStartAt` and `calendarEventId` to the Act type in `src/shared/types.ts`. The `SET_LINEUP` action from Claude should mark calendar-sourced acts as pinned.

### Files
- `src/shared/types.ts` — add `pinnedStartAt`, `calendarEventId` to Act type
- `src/renderer/views/WritersRoomView.tsx` — lineup rendering with pinned times
- `src/renderer/panels/LineupPanel.tsx` — lineup rendering
- `src/renderer/components/RundownBar.tsx` — rundown bar rendering
- `src/renderer/components/MiniRundownStrip.tsx` — mini strip rendering
