- [Medium] **Cross-boundary type import** — `HistoryView.tsx:5` imports types directly from `../../main/data/types`. While these are `import type` (erased at compile time, so no runtime coupling), this creates an architectural dependency from renderer → main. The types should live in `src/shared/types.ts` where the IPC channel constants already are, keeping the renderer → shared → main dependency direction clean. Currently the codebase has a single violation; if more renderer files start importing from main, it becomes a real problem.

- [Low] **DataService singleton uses module-scoped mutable state** — `DataService.ts:11` uses a module-level `let instance` for the singleton pattern. This works fine for Electron's single main process, but `initWithPath()` (line 41) creates a non-singleton instance without setting `instance`, meaning `getInstance()` won't return it. This is intentional (used in tests), but the dual path (singleton vs. non-singleton) in the same class mixes concerns. Not blocking — the test path is correctly scoped.

For a read-only history view this is fine (SQLite is fast, same connection, WAL mode), but if this pattern proliferates, consider a single joined query or a dedicated read model. Not a current problem.
